# iBut考试对策学习网页开发宪法

本文档从 `spec.md v1.0` 提炼开发规则。以后开发时，先检查本文件，再写代码。

## 1. 项目技术边界

1. 本项目只做静态前端页面。
2. 只允许使用 HTML、CSS、原生 JavaScript。
3. 不做后端。
4. 不做登录。
5. 不做数据库。
6. 不新增 spec.md 以外的功能。
7. 页面目标是帮助复习 iBut 考试对策资料。

## 2. HTML 规则

1. 必须使用语义化 HTML 标签。
2. 页面必须包含清晰的结构区域。
3. 顶部标题区使用 `header`。
4. 章节导航区使用 `nav`。
5. 主内容阅读区使用 `main`。
6. 独立内容块优先使用 `section` 或 `article`。
7. 页脚使用 `footer`。
8. 标题层级必须清楚，不要跳级乱用。
9. 表格只用于关键词中日英对照等表格数据。
10. 表单控件只用于搜索、筛选、练习题等真实交互。

## 3. CSS 规则

1. CSS 必须简单、清晰、容易修改。
2. 布局优先使用 FCC 已学过的 Flex 响应式布局。
3. 不使用 Tailwind。
4. 不使用 Bootstrap。
5. 不使用外部 UI 库。
6. 类名必须能看懂含义。
7. 相同样式可以复用 class。
8. 不写复杂到无法解释的选择器。
9. 页面在手机和电脑上都要能正常阅读。
10. 每个主要样式区域后面要有简单中文注释。

## 4. JavaScript 规则

1. JavaScript 只能使用原生写法。
2. 逻辑必须简单、可读、可逐行解释。
3. 不使用 React。
4. 不使用 Vue。
5. 不使用 Next.js。
6. 不使用任何框架语法。
7. 不写自己看不懂的高级写法。
8. 不使用 `_` 忽略逻辑。
9. 变量名必须表达真实含义。
10. 每个主要逻辑块后面要有简单中文注释。
11. JS 主要用于搜索、筛选、学习进度、模拟题交互。

## 5. 响应式规则

1. 所有页面必须适配移动端。
2. 手机宽度下内容不能横向溢出。
3. 章节目录和正文在小屏幕上要容易阅读。
4. 表格在小屏幕上必须可以正常查看。
5. 按钮、输入框、链接在手机上要方便点击。
6. 字体大小不能太小。
7. 内容块之间要有清楚间距。
8. 优先用 Flex 完成响应式布局。

## 6. 数据结构规则

1. 数据内容必须服务于 iBut 考试复习。
2. 章节数据要能对应页面章节导航。
3. 关键词数据要包含中文、日本語、English。
4. 题眼数据要帮助快速判断考试重点。
5. 模拟题数据要能对应当前章节。
6. 数据字段名必须简单、明确。
7. 不设计数据库结构。
8. 不加入登录用户数据。
9. 不加入后端接口数据。

## 7. Git 提交规则

1. 每次代码变更后必须给出 Git commit message。
2. commit message 要短、清楚、说明本次变更。
3. 文档变更使用 `docs:` 开头。
4. 功能变更使用 `feat:` 开头。
5. 修复变更使用 `fix:` 开头。
6. 不把无关文件混进同一次提交。
7. 提交前要确认没有修改 spec.md。

## 8. AI 协作规则

1. AI 必须严格遵守 spec.md。
2. AI 不得修改已锁定的 spec.md。
3. AI 不得新增 spec.md 以外的功能。
4. AI 写出的代码必须能让初学者逐行读懂。
5. AI 不能复制使用者看不懂的代码。
6. AI 每次改动前要先确认任务范围。
7. AI 每次改动后要说明改了什么。
8. AI 每次改动后要给出建议的 Git 命令。

## 9. 禁止事项

1. 禁止修改 spec.md。
2. 禁止新增后端。
3. 禁止新增登录功能。
4. 禁止新增数据库。
5. 禁止使用 React。
6. 禁止使用 Vue。
7. 禁止使用 Next.js。
8. 禁止使用 Tailwind。
9. 禁止使用 Bootstrap。
10. 禁止使用外部 UI 库。
11. 禁止写看不懂的高级代码。
12. 禁止使用 `_` 忽略逻辑。
13. 禁止让页面只适合电脑、不适合手机。
14. 禁止加入和 iBut 考试复习无关的功能。
